O que é Flask na prática

Flask é um microframework em Python para criar aplicações web. Neste ponto, a ideia já é dar sequência prática: abrir a pasta do projeto no editor, criar o app.py e preparar o terreno para o setup do próximo passo.

📁 Diretório

Pasta do projeto

Um caminho prático e fácil de localizar é c:\dev\flask\meu-projeto-flask. Abra exatamente essa pasta no editor antes de criar qualquer arquivo.

🧑‍💻 Editor

Editor de código

Pode ser VS Code ou PyCharm. O importante é abrir a pasta do projeto, enxergar os arquivos na lateral e então criar o app.py na raiz.

▶️ Continuidade

O que vem agora

Depois de abrir a pasta no editor e criar o arquivo inicial, o próximo passo é fazer o setup do ambiente para instalar dependências e rodar flask --app app run --debug.

Estrutura sugerida

c:\dev\flask\meu-projeto-flask\
└── app.py
📁Pasta enxuta e clara

Começar só com a pasta do projeto e o app.py ajuda o aluno a entender a base sem distração.

🚀Primeiro passo real

Mesmo simples, esse formato já prepara o terreno para crescer com venv, templates/, static/ e novas rotas.

Arquivo principal

app.py com Hello World

from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "Olá, Flask!"

if __name__ == "__main__":
    app.run(debug=True)

Primeiro setup do Flask

Aqui o ambiente deixa de ser abstrato e vira projeto de verdade. Neste momento, você já tem o app.py criado na raiz do projeto e agora vai preparar a mesma pasta para receber a venv, instalar o Flask e deixar tudo pronto para rodar a aplicação.

🧭 Ponto de partida deste passo

Antes de criar a venv, o aluno precisa entender que já está dentro da pasta do projeto e que o arquivo app.py já existe desde o passo anterior.

Isso é importante porque o ambiente virtual não nasce solto: ele faz parte da estrutura do projeto que você acabou de montar. Em outras palavras, primeiro existe a pasta do projeto com o app.py; depois, nessa mesma pasta, você cria a venv para isolar as dependências e continuar a configuração com organização.

Estrutura antes da venv

c:\dev\flask\meu-projeto-flask\
└── app.py

É dentro dessa pasta que o comando python -m venv venv deve ser executado.

Sequência guiada

🧪 Preparando o ambiente Flask passo a passo

Cada etapa fica separada para facilitar leitura, cópia e execução sem confusão. O fluxo correto aqui é: estar na pasta onde está o app.py, criar a venv, ativar o ambiente, instalar as dependências e então rodar o projeto.

Criar a venv

PASSO 4

Rotas básicas

Aqui começa o coração do Flask: o decorador @app.route. É ele que liga uma URL a uma função Python.

Ideia central

Quando alguém acessa uma rota como /, Flask executa a função associada e devolve uma resposta.

Uso típico

É o primeiro passo para criar páginas, APIs, painéis e endpoints internos.

Leitura da rota inicial

🏠Endereço inicial

O endereço / representa a página inicial da aplicação. É o ponto mais comum para começar um projeto web.

⚙️Função executada

A função home() é executada sempre que essa URL é acessada, ligando a rota ao comportamento em Python.

🚀Base do Flask

Mesmo sendo simples, esse padrão já ensina a base de quase tudo no Flask: rota, função, resposta e fluxo de navegação.

🧱Bloco reutilizável

Depois da primeira rota, você pode repetir essa mesma lógica para criar páginas como produto, contato, painel ou dashboard.

O início

Hello World

@app.route("/")
def home():
    return "Hello World"

Exemplo extra

Página de produto

@app.route("/produto")
def produto():
    return "Página de produto"
PASSO 5

Rotas com Parâmetros

Flask permite capturar partes da URL como variáveis. Isso é útil para perfis, produtos, categorias e páginas dinâmicas.

Onde isso aparece na prática

👤Páginas de perfil

Um exemplo clássico é a rota /usuario/ana, em que o nome vem direto da URL.

🛍️Detalhes de produtos

Você pode usar algo como /produto/15 para carregar o item certo com base no identificador.

🧩Recursos administrativos

Também funciona muito bem para painéis internos, edição por ID, slugs e áreas de gerenciamento.

URLs dinâmicas

Parâmetros na rota

Exemplo prático em c:\dev\flask\flask-exemplo

@app.route("/usuario/<nome>")
def usuario(nome):
    return f"Usuário: {nome}"

@app.route("/produto/<int:id>")
def produto(id):
    return f"Produto {id}"
PASSO 6

Rotas com Templates

Quando a rota precisa renderizar uma página HTML, usamos render_template com o motor Jinja2. Os arquivos devem ficar dentro da pasta templates/, sempre dentro da pasta principal do projeto.

Fluxo do template

🐍Python prepara os dados

A rota em Flask organiza as informações em Python antes de enviar qualquer coisa para a página.

🎨O HTML recebe com Jinja2

O template usa render_template para receber variáveis e montar a página com conteúdo dinâmico.

📁A pasta vem primeiro

No seu fluxo, o aluno precisa entender que a pasta do projeto vem primeiro e que a pasta templates/ fica dentro dela.

Estrutura esperada

c:\dev\flask\meu-projeto-flask\
├── app.py
└── templates\
    └── perfil.html

Ou seja: se o projeto estiver em c:\dev\flask\meu-projeto-flask, o template perfil.html deve ficar em c:\dev\flask\meu-projeto-flask\templates\perfil.html.

📍Onde o Flask procura

Por padrão, o Flask procura arquivos HTML dentro da pasta templates/. Se o arquivo ficar fora dela, a renderização não encontra a página.

Páginas web

render_template

from flask import render_template

@app.route("/perfil")
def perfil():
    return render_template("perfil.html", nome="Ana")

Lista dinâmica

Produtos

@app.route("/produtos")
def produtos():
    lista = ["Notebook", "Mouse", "Teclado"]
    return render_template("produtos.html", produtos=lista)

Dashboard

Resumo do sistema

@app.route("/dashboard")
def dashboard():
    dados = {"usuarios": 120, "pedidos": 38}
    return render_template("dashboard.html", dados=dados)
PASSO 7

Templates com Tailwind CDN

Para protótipos e treinamentos, você pode estilizar o HTML rapidamente adicionando o script do Tailwind CDN dentro do <head> do template.

Quando usar

⚡Prototipação rápida

O CDN é ótimo para estudo, validação visual rápida e páginas simples sem precisar configurar build.

🧪Treinamento e testes

Em aulas e exemplos, ele acelera muito porque o aluno consegue estilizar a página logo no primeiro contato.

Onde colocar

📄Template base

Inclua o script no template base, como base.html, para reaproveitar o estilo em várias páginas.

🧱Visão de projeto

Assim você centraliza layout, classes e estrutura principal em um único arquivo antes de criar páginas derivadas.

🔗 Link do CDN

🌐Script usado no projeto

https://cdn.tailwindcss.com

Coloque esse script no <head> para liberar as classes do Tailwind no template.

✨Facilidade no dia a dia

Com o CDN, você começa sem build nem instalação extra. Ótimo para aula, teste e protótipo.

📅Informação interessante

O Tailwind CSS foi lançado em 2017. Ficou popular por agilizar interfaces com classes utilitárias.

📁 Árvore do diretório

🗂️Onde fica o base.html

└── templates\
    └── base.html

O arquivo base.html fica dentro da pasta templates/, porque será renderizado pelo Flask.

🧭Por que isso ajuda

Com o base.html no lugar certo, você reaproveita cabeçalho, rodapé e o link do CDN em várias páginas.

PASSO 8

Rotas com Métodos HTTP

Uma mesma rota pode responder de forma diferente conforme o método HTTP, como GET, POST, PUT ou DELETE.

Leitura rápida dos métodos

📖GET

Costuma exibir dados, páginas, listas e detalhes que o usuário quer consultar.

📨POST

Envia dados novos, como cadastro, contato, login ou criação de registros.

🛠️PUT

É usado para atualizar informações já existentes em um recurso da aplicação.

🗑️DELETE

Serve para remover dados, registros ou recursos que não serão mais usados.

Separar bem esses métodos ajuda a deixar o fluxo da aplicação mais previsível, organizado e profissional.

Formulários e ações

GET e POST

from flask import request

@app.route("/contato", methods=["GET", "POST"])
def contato():
    if request.method == "POST":
        return "Formulário enviado"
    return "Página do formulário"
PASSO 9

Rotas de API com JSON

Quando a rota vai servir dados para FlutterFlow, React, mobile ou outro frontend, o retorno ideal costuma ser JSON com jsonify.

Boas práticas para resposta JSON

🏷️Chaves claras

Use nomes de chaves simples e previsíveis, como status, dados e erro.

🧱Estrutura estável

Mantenha o mesmo formato de resposta para facilitar o consumo no frontend e reduzir confusão.

📈Crescimento saudável

Padronizar cedo ajuda muito quando a API começa a crescer e passa a ter mais rotas e clientes.

API

jsonify

from flask import jsonify

@app.route("/api/produtos")
def produtos_api():
    return jsonify({
        "status": "ok",
        "dados": [
            {"id": 1, "nome": "Notebook"},
            {"id": 2, "nome": "Mouse"}
        ]
    })

Bibliotecas muito usadas com Python Flask

Essas bibliotecas aparecem com frequência em APIs, autenticação, banco de dados, validação e deploy.

1. Base

Flask

Framework principal para criar rotas, APIs, páginas e a estrutura inicial do projeto.

2. Banco

Flask-SQLAlchemy

ORM para banco de dados, ideal para modelos, consultas e relacionamento entre tabelas.

3. Estrutura

Flask-Migrate

Gerencia migrações de banco para manter a estrutura sincronizada com os modelos.

4. Login

Flask-Login

Autenticação de sessão para áreas logadas, perfis de usuário e controle de acesso.

5. API

Flask-JWT-Extended

JWT para APIs modernas com autenticação por token e proteção de endpoints.

6. Ambiente

python-dotenv

Carrega variáveis de ambiente do arquivo .env para deixar o projeto mais seguro.

7. Consumo

requests

Consome APIs externas para integrar serviços, buscar dados e automatizar fluxos.

8. Validação

marshmallow

Serialização e validação para deixar payloads e respostas mais consistentes.

9. Deploy

gunicorn

Servidor de aplicação comum em deploy, especialmente em ambientes Linux.

10. Testes

pytest

Testes automatizados para validar rotas, funções e comportamentos importantes.
