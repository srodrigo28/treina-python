Git e GitHub do Zero

Módulo Inicial

Git do Zero Absoluto

Entenda como salvar o histórico do seu código e trabalhar em nuvem. Vamos configurar o Git na sua máquina e criar sua primeira conta no GitHub.

Git (Local)

É a ferramenta instalada no seu PC. Ela cria uma "máquina do tempo" para o seu código, permitindo salvar versões e desfazer erros.

GitHub (Nuvem)

É um site onde você hospeda seus projetos Git. Serve como backup, portfólio e ferramenta de trabalho em equipe.

PASSO 1

Instalação e Identidade

O Git precisa saber quem você é antes de salvar qualquer código.

O que isso resolve

Instala a ferramenta que cria o histórico do seu projeto e configura sua identidade para que cada commit fique associado ao seu nome e e-mail.

Por que isso importa

Sem Git instalado e configurado, você não consegue versionar arquivos, enviar mudanças para o GitHub nem manter um histórico confiável do trabalho.

1. Baixar e Instalar

Baixe o instalador oficial do Git para Windows antes de continuar:

Abrir download oficial do Git
Windows
Download direto do instalador 64-bit para Windows.
Linux (Ubuntu)
sudo apt install git -y

2. Configurar Identidade

Abra seu terminal (ou Git Bash no Windows) e execute. Isso define a assinatura dos seus commits.

Terminal 
git config --global user.name "Seu Nome Aqui"
git config --global user.email "seuemail@exemplo.com"

# Verificar se deu certo:
git config --list
PASSO 2

Criando a Casa do Código

Configurando sua conta e criando o primeiro repositório remoto.

O que isso resolve

Cria o local online onde seu projeto ficará hospedado, permitindo backup, compartilhamento e colaboração.

Por que isso importa

Sem um repositório remoto, seu código fica preso à sua máquina e você perde a parte mais útil do GitHub: publicar, versionar em nuvem e trabalhar em equipe.

1

Criar Conta

Acesse github.com e faça seu cadastro gratuito (Sign Up). Use o mesmo email que configurou no terminal.

2

Novo Repositório (New Repo)

No canto superior direito, clique no + e selecione New repository.

  • Repository name: meu-primeiro-projeto
  • Public/Private: Public
  • Deixe as outras opções desmarcadas por enquanto.
  • Clique em Create repository.
3

Copiar o Link HTTPS

Após criar, o GitHub mostrará uma tela de setup. Copie o link na área "HTTPS".

https://github.com/SEU-USUARIO/meu-primeiro-projeto.git
PASSO 3

Trabalhando Localmente

Os 3 comandos sagrados que você usará todos os dias.

O que isso resolve

Ensina o fluxo básico para transformar alterações em versões salvas: iniciar, preparar arquivos e registrar um commit.

Por que isso importa

Esse é o núcleo do Git. Se você dominar esse fluxo, já consegue trabalhar com segurança sem perder alterações importantes.

1. Iniciar o Projeto

Navegue até a pasta do seu projeto e transforme-a em um repositório Git.

git init

2. Adicionar Arquivos (Stage)

Prepara os arquivos alterados para serem salvos. O . seleciona tudo.

git add .

3. Salvar Versão (Commit)

Cria um ponto na história com uma mensagem descrevendo o que foi feito.

git commit -m "Meu primeiro commit"
PASSO 4

Conectando ao Mundo

Vinculando seu Git local ao GitHub e enviando os arquivos.

O que isso resolve

Liga seu projeto local ao repositório do GitHub e publica a primeira versão online com o comando de push.

Por que isso importa

É aqui que o projeto deixa de existir só no seu computador e passa a ficar salvo, acessível e pronto para colaboração.

Terminal 
# 1. Renomear a branch principal para 'main' (padrão atual)
git branch -M main

# 2. Adicionar o link do repositório (Cole o link do Passo 2)
git remote add origin https://github.com/SEU-USUARIO/meu-primeiro-projeto.git

# 3. Enviar os arquivos (Push)
git push -u origin main

Atenção: Ao fazer o primeiro push, o GitHub pedirá login.

Se a senha não funcionar, você precisará gerar um Token de Acesso nas configurações do GitHub ou usar o "Sign in with Browser" se estiver usando o Git for Windows.

Resumo do Dia a Dia

Depois de tudo configurado, seu fluxo será apenas:

1. git add .
2. git commit -m "mensagem"
3. git push
