Bem-vindo

ReactJS: A Biblioteca do Futuro

React é uma biblioteca JavaScript para criar interfaces de usuário. Diferente de sites antigos, ele cria Single Page Applications (SPA), onde a página nunca recarrega, apenas o conteúdo muda.

Baseado em Componentes

Você constrói pequenos blocos (Botão, Header, Card) e os monta como Lego para criar sites complexos.

Rápido (Vite)

Antigamente usávamos o CRA (Create React App). Hoje usamos Vite, que é instantâneo e leve.

PASSO 1

Verificando o Motor

O React precisa do Node.js instalado no seu computador para rodar o ambiente de desenvolvimento.

Teste de Instalação

Abra seu terminal e digite:

Terminal 
node -v
npm -v

Saída Esperada

Se aparecer números como v18.16.0 ou superior, você está pronto.

Deu erro?

Baixe a versão LTS em nodejs.org.

PASSO 2

Criando o Projeto

Vamos usar o Vite. Ele é muito mais rápido que o antigo create-react-app.

Opção A: Projeto com JavaScript (JSX)

Ideal para quem está aprendendo a lógica do React sem se preocupar com tipagem.

Terminal 
npm create vite@latest meu-projeto-js -- --template react
cd meu-projeto-js
npm install
npm run dev

Opção B: Projeto com TypeScript (TSX)

O padrão profissional de mercado. Mais seguro e com autocompletar inteligente.

Terminal 
npm create vite@latest meu-projeto-ts -- --template react-ts
cd meu-projeto-ts
npm install
npm run dev
PASSO 3

Anatomia do Projeto

Entendendo onde cada coisa vive.

  • node_modules/ Bibliotecas (não mexa)
  • public/ Imagens estáticas
  • src/
  • main.jsx (ou .tsx) Ponto de entrada
  • App.jsx (ou .tsx) Componente Principal
  • index.css Estilos globais
  • package.json Config e Scripts
  • vite.config.js Config do Vite

main.jsx

É onde o React "nasce". Ele pega o elemento #root do HTML e injeta o App inteiro dentro dele.

App.jsx

É a sua "página inicial". Tudo o que você vê na tela começa sendo montado aqui.

package.json

A "identidade" do projeto. Lista quais bibliotecas você instalou e quais comandos (scripts) pode rodar.

PASSO 4

A Batalha das Extensões

Qual a diferença real entre programar em .jsx e .tsx?

JSX (JavaScript XML)

Flexível 
function Botao({ texto }) {
  return (
    <button>
      {texto}
    </button>
  );
}

// Uso (Risco de erro):
<Botao texto={123} /> 
// Funciona, mas pode quebrar o layout

Prós: Menos código para escrever. Rápido para protótipos.
Contras: Você não sabe o que o componente espera receber até dar erro.

TSX (TypeScript XML)

Seguro 
interface Props {
  texto: string;
}

function Botao({ texto }: Props) {
  return (
    <button>
      {texto}
    </button>
  );
}

// Uso (Segurança):
<Botao texto={123} /> 
// ERRO IMEDIATO: "Number is not string"

Prós: Autocomplete perfeito. O editor avisa erros antes de rodar.
Contras: Mais verboso (tem que criar interfaces).
