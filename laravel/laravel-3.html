Segurança & API

Vamos criar uma nova aplicação focada em Autenticação. Usaremos o Laravel Breeze para gerar telas de login em segundos e depois criaremos uma API segura.

Importante

Não use o projeto do Módulo 2. O Breeze substitui arquivos e pode quebrar seu CRUD manual. Vamos começar do zero para garantir um ambiente limpo.

Criando o Projeto

Vamos gerar uma nova aplicação Laravel limpa.

1. Terminal

Terminal 
# Volte para sua pasta de projetos (ex: htdocs ou www)
cd .. 

# Crie um novo projeto chamado 'laravel-auth'
composer create-project laravel/laravel laravel-auth

# Entre na pasta
cd laravel-auth

2. Banco de Dados

Configure o arquivo .env com um banco novo (ex: laravel_auth_db). Não esqueça de criar esse banco no PHPMyAdmin.

DB_DATABASE=laravel_auth_db
DB_USERNAME=root
DB_PASSWORD=

Instalando Laravel Breeze

Breeze instala rotas, controllers e views para Login, Registro e Recuperação de Senha.

1. Requerer o Pacote

composer require laravel/breeze --dev

2. Instalar e Compilar

Este comando vai criar as views e pedir para instalar as dependências de Front-end.

# Instala o scaffolding usando Blade (padrão)
php artisan breeze:install blade

# Cria as tabelas de usuário e sessão no banco
php artisan migrate

# Instala dependências CSS/JS (Node.js necessário)
npm install
npm run dev

Teste Agora!

Abra http://127.0.0.1:8000/login. Você verá um sistema de login completo e funcional.

Criando uma API JSON

Laravel separa rotas web (com sessão/cookies) de rotas API (sem estado/tokens) no arquivo routes/api.php.

1. Controller de API

php artisan make:controller Api/TaskApiController --api

A flag --api cria um controller limpo, sem métodos `create` ou `edit` (que retornam HTML).

2. Definindo a Rota (routes/api.php)

use App\Http\Controllers\Api\TaskApiController;

// O Laravel adiciona o prefixo /api automaticamente
Route::apiResource('tasks', TaskApiController::class);

API Resources

Transformando Models em JSON limpo e seguro.

1. Criar Resource

php artisan make:resource TaskResource

2. Configurar (app/Http/Resources/TaskResource.php)

public function toArray(Request $request): array
{
    return [
        'id' => $this->id,
        'titulo' => $this->name, // Mudando nome do campo para quem consome
        'categoria' => $this->category,
        'concluido' => (bool) $this->status,
        'criado_em' => $this->created_at->format('d/m/Y'),
    ];
}

3. Usar no Controller

public function index() {
    // Retorna a coleção formatada pelo Resource
    return TaskResource::collection(Task::all());
}

Tokens com Sanctum

Laravel Sanctum já vem instalado e oferece um sistema leve de autenticação via Token.

Criar Rota de Login API

No controller Api/AuthController.php (crie este arquivo):

public function login(Request $request) {
    $user = User::where('email', $request->email)->first();

    if (! $user || ! Hash::check($request->password, $user->password)) {
        return response()->json(['msg' => 'Falha'], 401);
    }

    // Gera o token de acesso
    $token = $user->createToken('auth_token')->plainTextToken;

    return response()->json(['token' => $token]);
}

Protegendo Rotas API

Use Route::middleware('auth:sanctum') no arquivo api.php para exigir o token nas rotas de tarefas.
